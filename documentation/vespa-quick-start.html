This article is tested using Docker for Mac - adapt as fit for other platforms. Also try the CentOS quick-start.

  1. Clone vespa git repository on github.com: 

    $ git clone git@github.com:vespa-engine/vespa.git
    We assume that the vespa source repository is available in an env variable $VESPA_SOURCE, e.g VESPA_SOURCE=/Users/foo/github/vespa.

  2. Install Docker for Mac: Let Docker use 4GB in Preferences

  3. Start a Vespa Docker container: 

    $ docker run --detach --name vespa --hostname vespa-container --privileged --volume $VESPA_SOURCE:/vespa-source --publish 8080:8080 vespaengine/vespa
    We let the previously downloaded source code be available inside the docker container filessystem as /vespa-source using the docker volume option. We also publish port 8080 for allowing search and feed interface to be accessable outside of the docker container.

  4. Log into the Vespa Docker container: Using the vespa container name from the above docker creation command we can execute commands inside the container, e.g start a bash shell by 

     $ docker exec -it vespa bash

  5. Deploy and activate a sample application: Sample applications in sample-apps. Read more on applications in Application packages: 

    [root@vespa-container #/] vespa-deploy prepare /vespa-source/sample-apps/basic-search/src/main/application/ 
[root@vespa-container #/] vespa-deploy activate

  6. Ensure application is active: Wait for a valid 200 OK response by: 

    [root@vespa-container #/] curl -s --head http://localhost:8080/ApplicationStatus

  7. Feed documents: Read more on feeding and retrieving documents in the Document API: 

    [root@vespa-container #/] curl -s -X POST --data-binary @/vespa-source/sample-apps/basic-search/music-data-1.json http://localhost:8080/document/v1/music/music/docid/1 |python -m json.tool
[root@vespa-container #/] curl -s -X POST --data-binary @/vespa-source/sample-apps/basic-search/music-data-2.json http://localhost:8080/document/v1/music/music/docid/2 |python -m json.tool

  8. Run a search query and a single get request : 

    [root@vespa-container #/] curl -s http://localhost:8080/search/?query=bad| python -m json.tool 
[root@vespa-container #/] curl -s http://localhost:8080/document/v1/music/music/docid/2 |python -m json.tool
    Since we published port 8080 outside of the docker container you can also view the results in your favorite browser at http://localhost:8080/search/?query=bad. Read more in the Search API:
