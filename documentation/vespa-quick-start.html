This article is tested using Docker for Mac - adapt as fit for other platforms. Also try the CentOS quick-start.

  1. Install Docker for Mac: Let Docker use 4GB in Preferences

  2. Build or load a Vespa Docker image:

    1. Use instructions at Building Vespa Docker image.

    2. Or load an already built image using: 

      $ docker load -i <Vespa docker image>.tar

  3. Start a Vespa Docker container: Use instructions at Running Vespa inside Docker container. Start one container per application.

  4. Locate the container id: 

    $ docker ps

  5. Log into the container: 

    $ docker exec -it <container id> bash

  6. Deploy and activate a sample application: Sample applications in sample-apps. Read more on applications in Application packages: 

    $ cd /vespa/sample-apps/basic-search
$ vespa-deploy prepare src/main/application && vespa-deploy activate

  7. Ensure application is active: Wait for a valid JSON reply with info about the active application: 

    $ curl "localhost:8080/ApplicationStatus"

  8. Feed documents: Read more on feeding and retrieving documents in the Document API: 

    $ curl -X POST --data-binary @music-data-1.json "localhost:8080/document/v1/music/music/docid/1"
$ curl -X POST --data-binary @music-data-2.json "localhost:8080/document/v1/music/music/docid/2"

  9. Run a query: Read more in the Search API: 

    $ curl "localhost:8080/search/?query=michael"

Next steps:

  • Try the Blog search tutorial to learn more about using different Vespa features.
  • Developing with Vespa means writing Java plugins for document and query processors. Try developing applications to compile and test Vespa JDisc Container plugins.
  • Vespa APIs is useful to understand how to interface with Vespa.
  • Explore the sample applications.
  • Ensure that you know how to secure your Vespa installation.
