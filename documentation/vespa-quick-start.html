This article is tested using Docker for Mac — adapt as you see fit for other platforms. Alternatively, try the CentOS quick-start.

  1. Clone vespa sample applications git repository on github.com: 

    $ git clone git@github.com:vespa-engine/sample-apps.git
$ export VESPA_SAMPLEAPP=/Users/foo/github/sample-apps     # replace with correct path

  2. Install Docker for Mac: Let Docker use 4GB in Preferences

  3. Start a Vespa Docker container: 

    $ docker run --detach --name vespa --hostname vespa-container --privileged \
  --volume $VESPA_SAMPLEAPP:/vespa-sampleapp --publish 8080:8080 vespaengine/vespa
    The volume option makes the previously downloaded source code available, as /vespa-source, inside the docker container. We also publish port 8080 to make the search and feed interfaces accessible outside of the docker container.

  4. Log in to the Vespa Docker container: Using the vespa container name from the above docker run command, we can execute commands inside the container, e.g., start a bash shell with: 

    $ docker exec -it vespa bash

  5. Deploy and activate a sample application: 

    [root@vespa-container #/] vespa-deploy prepare /vespa-sampleapp/basic-search/src/main/application/ && vespa-deploy activate
    More sample applications can be found in sample-apps. Read more on applications in Application packages.

  6. Ensure the application is active: Wait for a valid 200 OK response from: 

    [root@vespa-container #/] curl -s --head http://localhost:8080/ApplicationStatus

  7. Feed documents: 

    [root@vespa-container #/] curl -s -X POST --data-binary @/vespa-sampleapp/basic-search/music-data-1.json \
                          http://localhost:8080/document/v1/music/music/docid/1 | python -m json.tool
[root@vespa-container #/] curl -s -X POST --data-binary @/vespa-sampleapp/basic-search/music-data-2.json \
                          http://localhost:8080/document/v1/music/music/docid/2 | python -m json.tool
    Read more on feeding and retrieving documents in the Document API.

  8. Run a search query and a single get request : 

    [root@vespa-container #/] curl -s http://localhost:8080/search/?query=bad | python -m json.tool 
[root@vespa-container #/] curl -s http://localhost:8080/document/v1/music/music/docid/2 | python -m json.tool
    Since we published port 8080 outside of the docker container, you can also view the results in a browser at http://localhost:8080/search/?query=bad. Read more in the Search API.

Next steps:

  • Try the blog search tutorial to learn more about the different Vespa features.
  • Developing with Vespa includes writing Java plugins for document and query processors; try developing applications to compile and test Vespa JDisc Container plugins.
  • The Vespa APIs are useful to understand how to interface with Vespa.
  • Explore the sample applications.
  • Ensure that you know how to secure your Vespa installation.
